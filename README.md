# realtime-meeting-transcriber

日本語・英語・中国語が混在するWeb会議を、**ローカル完結（ゼロ円）**で
リアルタイム文字起こし＆日本語訳し、離席中・会議後に要約するツール。

## 特徴
- 相手の声（スピーカー出力）を WASAPI ループバックで取得
- 自分の声（マイク入力）も同時取得し、発言を**話者ラベル（自分／相手）付き**で記録
- faster-whisper (large-v3) で文字起こし、言語を自動判定
- 非日本語は qwen2.5:14b（Ollama）でローカル翻訳
- タイムスタンプは**録音時刻ベース**なので、処理が詰まっても時間軸がずれない
- 「離席 → 復帰」で離席中だけを要約／会議後に全体要約
- マイクが無い環境では相手の声のみで自動的に続行
- 外部API不要・追加課金なし

## 構成
```
会議の音声
  相手: スピーカー出力 ──┐
  自分: マイク入力 ──────┤
                         ↓
  → audio.py        WASAPI で2系統を取り込み・16kHz化・録音時刻を付与
  → transcriber.py  silero-VAD で発話を切り出し → faster-whisper
                    → 非日本語は qwen2.5:14b で日本語訳（話者ラベル付き）
  → run.py          タイムスタンプ付きログ／離席要約／会議後要約
```

## 必要環境
- Windows 10 (1903以降) … WASAPIループバック
- NVIDIA GPU + CUDA … faster-whisper large-v3（VRAM 12GB+目安）
- Ollama + `qwen2.5:14b`
- Python 3.11

## セットアップ
```powershell
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```
Ollama を起動し、`ollama pull qwen2.5:14b` 済みであること。

## 使い方
ダブルクリックで起動（推奨）。`start.bat` をプロジェクト直下に置く。
Ollama の起動確認・venv 有効化・文字コード設定をまとめて実行する。
```
start.bat
```

手動で起動する場合:
```powershell
venv\Scripts\activate
python -m meeting.run
```

| コマンド | 動作 |
|---------|------|
| a | 離席を記録 |
| b | 復帰（離席中を要約） |
| s | 直近5分を要約 |
| f | 会議全体を要約して保存 |
| q | 終了（全体要約を保存） |

保存済みログから要約を作り直す:
```powershell
python -m meeting.summarize_file
```

## デバイス選択
取り込むデバイスは自動判定するが、Bluetooth など意図しないデバイスを掴む場合は、
一覧を確認して名前で明示指定できる。

デバイス一覧を表示:
```powershell
python -m meeting.audio
```

環境変数で対象デバイスを名前指定（部分一致）:
```powershell
$env:LOOPBACK_HINT="ヘッドセット"   # 相手の声（出力ループバック）
$env:MIC_HINT="ヘッドセット"        # 自分の声（マイク入力）
python -m meeting.run
```

## ログの保存先
`data/meetings/` に保存される（`.gitignore` 対象）。
実データを含むためコミットしないこと。

## 既知の制約 / まだ解決していないこと
- **Bluetooth のプロファイル制約**：BTイヤホンは「相手＝A2DP（高音質）」と
  「自分＝HFP（マイク）」を同時に両立できない。通話でマイクが有効になると出力も
  HFP（16kHz モノラル）へ切り替わり、ループバック対象がずれることがある。
  確実に録るなら出力を有線/スピーカー等の非BTにするのが安定。
- **翻訳が同期実行**：翻訳中（最大数十秒）は表示が一時的に止まる。ただしタイムスタンプは
  録音時刻ベースなので、ログの時刻・順序は乱れない（非同期化は今後）。
- **言語混在の誤判定**：話者が外国語の専門用語を挟むと言語判定を誤ることがある。
- **同音異義語の誤変換**：固有名詞（例：京都／恐怖）を取り違えることがある。
- **相手側の話者分離なし**：自分／相手は区別するが、相手が複数いる場合の「誰が」は未対応。
